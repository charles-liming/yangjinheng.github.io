发表于 2019-06-13 | 分类于 golang

OS 包介绍

Go 语言标准库 os 包提供了不依赖平台的操作系统接口

设计为 Unix 风格的，而错误处理是 Go 风格的，失败的调用会返回错误值而非错误码，通常错误值里包含更多信息。

  1. OS 包的功能

exec

exec 包执行外部命令。它包装了os.StartProcess函数以便更容易的修正输入和输出，使用管道连接I/O，以及作其它的一些调整。

Cmd 对象

Cmd 代表一个正在准备或者在执行中的外部命令。

command

Command 方法返回 Cmd struct 以使用给定参数执行命名程序，返回值只设定了 Cmd 的 Path 和 Args 两个参数，如果 name 不含路径分隔符，则尝试使用 LookPath 获取完整路径，否则直接使用 name 作为路径，name 始终解析为命令的名称，arg 始终是参数。

文件流读写

Reader

输入流

流（stream）是应用程序和外部资源进行数据交互的纽带

流分为输入流和输出流，输出和输出都是相对于程序，把外部数据传入到程序中叫做输入，反之叫输出

输入流（input stream）输出流（output stream）就是平时所说的 I/O 流

在 Go 语言标准库 io 包下是 Reader 接口表示输入流，只要实现这个接口就属于输入流

  • 创建字符流

打开文件流

Writer

输出流就是把程序中的数据写出到外部资源，Go 语言标准库中输出流是 Writer 接口

注意：输出流的时候，不要使用 os.Open() 因为这种方式获取的文件是只读的

ioutil

ioutil 包下提供了对文件读写的工具函数，通过这些函数快速实现文件的读写操作

ioutil 包下提供的函数比较少，但是都很方便的函数
