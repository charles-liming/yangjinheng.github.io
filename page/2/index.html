注意事项

  1. Go 源码是以 .go 为扩展名的
  2. Go 应用程序的执行入口是 main() 函数
  3. 严格区分大小写
  4. Go 代码的每个语句结尾不需要加 ;
  5. Go 编译器是一行一行编译的，因此一行就写一条语句，否则报错
  6. Go 语言 import 包如果没有使用到，那么代码不能编译通过

切片的英文名称 slice，切片是具有可变长度，相同类型元素的序列，由于长度是可变的，可以解决数组长度在数据不确定的情况下浪费内存的问题。

  • 声明语法，和数组的唯一区别是不需要声明长度
  • 仅声明切片时，没有开辟内存空间，不能直接对切片进行操作，需要初始化

make初始化切片

切片的容量与长度

长度是切片元素个数，容量切片可以有的元素个数。

长度可可以使用索引访问，超出索引的容量还没有元素，所以不能使用索引访问。

  • 使用 make 函数定义的切片没有内容，但这个切片已经申请了内存空间。

通过数组产生切片

  • 取出数组的一部分就会产生一个切片
  • 切片是指针，指向数组元素地址，修改切片的内容，数组的内容会发生变化
  • 当向数组产生的切片添加内容时，切片增加的部分实际是在修改原数组的内容
  • 当切片超出数组长度时候，超出部分会被分配新的内存空间，同时内存地址发生变动

  • 基础语法
  • copy 函数会严格按照索引进行从源到目标的复制，
  • 源数据多余目标所能承载的部分，则会 copy 函数会抛弃多余的部分。
  • 也可以复制一部分

包的组织

Go 语言中组织单元最大的为项目，项目由多个包组成，包由多个文件组成，文件包含了很多行代码

包在文件系统中表现在一个目录，目录下所有的的文件中的 package 必须相同，建议与目录同名

包的规范

  1. 包是 Go 语言中的逻辑分组，物理上表现为不同的目录。
  2. 目录下所有文件都属于一个包，且文件中定义的 package 关键字必须一致。
  3. 虽然允许源码文件的 package 和目录名称不同，但是最终编译都会编译成目录名称。
  4. 一个 Go 语言项目必须要有 main 包，其他包个的个数不限。
  5. Go 语言在寻找包的时候会从 GOPATH/src 路径中寻找包，如果不存在，则会去 GOROOT 目录中寻找。
  6. 不同包资源下资源可以互相访问，在导入其他包之后，可以访问包下首字母为大写的内容。
  7. 包下不同文件中全局资源可以随意访问。

GC

GC 英文名称全程：gtbage collector，一些常用的 GC 算法：

  1. 引用计算法

当对象被引用时计数器增加一，不被引用时计数器减少一，PHP 和 Object-C 使用，缺点是：互相引用的无法回收，计数器增加消耗。

  1. 标记清除

停止程序运行，遍历对象，进行标记，标记完成后将没有引用的对象进行清除，缺点是：由于标记清除需要停止程序，当对象特别多的时候，标记和清除的过程比较耗时

  1. 三色标记法

是标记清除的改进版，从逻辑上分为白色区域（未搜索）、灰色区域（正在搜索）、黑色区域（已经搜索）灰色区域内容是子引用没有进行搜索，黑色区域表示子引用已经存在

  1. 分代收集

一般有三代，例如：Java 新生代、老年代、永久代，当新生代中达到阈值时会把对象放入老年代，相同道理当老年代内容达到阈值，会放入永久代

Go 语言的 GC

  1. Go 语言采用 Stop The World 方式，就是三色标记法
  2. 每个 Go 语言版本基本上都会对 GC 进行优化，从 Go 语言 1.5 开始支持并发收集，从 1.8 版本已经把 STW 时间优化到了 100 微秒，通常只需要 10 微秒以下，且在 1.10 版本时再次优化 GC 对 CPU 的占用
  3. Go 语言中 GC 是自动运行的，在下列情况下会触发 GC

当需要申请内存时，发现 GC 是上次 GC 两倍时会触发

每 2 分钟自动运行一次 GC

  1. GC 调优，小对象复用、局部变量尽量少声明，多个小对象可以放入结构体，方便 GC 扫描，少用字符串连接
  2. 在 runtime 包下 mgc.go 中明确的说明了 Go 语言的 GC 的解释
