NFS-Client

nfs-client 是一个自动配置程序，它使用现有的和已配置的 NFS服务器来支持通过持久卷声明动态配置Kubernetes持久卷。

  • 安装 NFS

  • 准备目录
  • 启动 NFS 并设置开机启动
  • 在所有的 node 节点安装 rpcbind

部署NFS-Client

  • 部署 rbac.yaml 安装 NFS-Client 所需的 Service Account
  • 部署 deployment.yaml 到 K8S 集群，修改 yaml 文件中的NFS服务器所在的IP地址以及NFS服务器共享的路径，指定 PROVISIONER_NAME 为自己需要的 StorageClass 名字
  • 创建 StorageClass，注意修改 StorageClass 的名字与 nfs-client-provisioner 中的 PROVISIONER_NAME 一致

使用Ceph-RBD-实现动态卷

发表于 2019-05-12 | 分类于 Kubernetes

StorageClass Ceph RBD

实验走通了就懒得记笔记了，简单说下过程：

  1. 搭建一个 Ceph 集群
  2. 在 Ceph 集群中创建一个存储池，供 K8S 的 StorageClass 来申请
  3. 部署 rbd-provisioner（自1.12版本之后必须安装的插件），StorageClass 需要调用它产生实例
  4. 新建 StorageClass 访问 Ceph 的 Secret
  5. 新建 StorageClass 并将 provisioner 指向 ceph.com/rbd
  6. 新建 PersistentVolumeClaim 声明请求存储的大小类型等
  7. 在 POD 中使用 PersistentVolumeClaim 类型的 Volume，并在容器中挂载

配置 Ceph 储存池

  • 创建 ceph 存储池

安装 rbd-provisioner

  • 1.12 版本后 kube-controller-manager 不再内置 rbd 命令，所以 StorageClass 的 provisioner 而是通过外部的插件来实现

使用 StorageClass

  • 创建 CephX 验证 secret
  • 创建 StorageClass 指向 rbd-provisioner，
  • 创建 PersistentVolumeClaim
  • 在 POD 中使用 PVC，最后在容器中挂载 PVC。
